KpOS modular packages patch
===========================

Added:
- ios/lib/package_manager.lua
  Discovers packages from ios/packages/<id>/package.lua or manifest.lua.

- ios/packages/README.txt
  Documents the package folder structure and manifest fields.

- ios/packages/minechat/package.lua
  Wraps the existing Minechat app as a modular package.

- ios/pkg.lua
  CLI helper:
    ios/pkg.lua list
    ios/pkg.lua info <id-or-name>
    ios/pkg.lua run <id-or-name>

Changed:
- ios/.programs.lua
  Replaced the hard-coded Programs menu with dynamic package discovery.

Fixed:
- ios/.programs.lua no longer uses invalid Lua 'elif'.
- ios/.update.lua now calls shell.run("exit.lua") correctly.
- ios/programs/Minechat/chat_client.lua /quit now exits properly.
- test.lua no longer has the invalid keys. expression.

Safety:
- .minechat_session was not included in the patched zip because it contained a saved login token.

Hotfix:
- Fixed ComputerCraft's program-local shell API not being visible inside ios/lib/package_manager.lua when launching packages.
- PackageManager.runPackage can now receive shell from the menu/CLI and falls back to os.run when shell is unavailable.
